# Internationalization (i18n)

## User-Facing Strings

- ALL user-facing text must use `t("dotted.key.path")` — no hardcoded strings
- Translation function supports nested keys: `t("labels.paste")`, `t("toolBar.rectangle")`
- For dynamic values use `{{placeholder}}` syntax: `t("key", { count: 5 })`

## Adding Translations

- Add new keys to `packages/excalidraw/locales/en.json` FIRST (English is the fallback)
- Other languages will fall back to English for missing keys
- Translations are loaded lazily via dynamic imports

## React Integration

- Use `useI18n()` hook in components that need to react to language changes
- The hook uses Jotai atom (`editorLangCodeAtom`) for efficient re-renders
- Components receiving `langCode` via props do NOT need the hook

## RTL Support

- Some languages are RTL — never assume LTR layout in new components
- Use logical CSS properties (`margin-inline-start`) over physical (`margin-left`) where applicable

## How to Verify

- Grep for hardcoded user-facing strings in changed/new components — should use `t()` instead
- Check that new keys exist in `packages/excalidraw/locales/en.json`
- Run `yarn test:update` — i18n-related tests pass
